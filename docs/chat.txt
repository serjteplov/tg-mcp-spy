  Getting API_ID and API_HASH

  1. Go to https://my.telegram.org
  2. Log in with your phone number
  3. Go to API development tools
  4. Create an application (fill in any name/description вҖ” it doesn't matter for
  personal use)
  5. Copy the App api_id and App api_hash

  Getting SESSION_STRING

  This is the tricky part. You need to generate a StringSession by logging in
  interactively once. The README notes it must be "generated externally." The
  typical way:

  1. Install Telethon in a temporary environment: pip install telethon
  2. Run this short script:

  from telethon.sync import TelegramClient
  from telethon.sessions import StringSession

  api_id = 123456          # your API_ID
  api_hash = "your_hash"   # your API_HASH

  with TelegramClient(StringSession(), api_id, api_hash) as client:
      print(client.session.save())

  3. It will prompt you for your phone number and the login code Telegram sends
  you
  4. If you have 2FA enabled, it will also ask for your password
  5. The printed string is your TELEGRAM_SESSION_STRING вҖ” save it and never 
  share it (it grants full access to your account)

  в–Һ вҡ пёҸ Security: The session string is equivalent to being logged into your 
  в–Һ account. Treat it like a password. Never commit it to git.

  ---
  2. How to launch the server

  # Install dependencies
  uv sync --all-extras

  # Set required env vars
  export TELEGRAM_API_ID=your_api_id
  export TELEGRAM_API_HASH=your_api_hash
  export TELEGRAM_SESSION_STRING=your_session_string

  # Run the server
  python -m package_tgmcpspy.server

  Or use the installed entry point:

  tg-mcp-spy

  The server binds to 127.0.0.1:8000 by default.

  On startup, it validates the session вҖ” if the session is invalid or expired,
  it fails immediately with a clear authorization error (per spec requirement
  R3).

  ---
  4. Where the SQLite database is configured

  The TGMCPSPY_DB_PATH env var controls where the SQLite database file is
  created:

  - Default: tgmcpspy.db (a file in the current working directory where you
  launch the server)
  - Custom: set TGMCPSPY_DB_PATH to any path, e.g. export 
  TGMCPSPY_DB_PATH=/home/serg/.local/share/tgmcpspy/cache.db

  The database is auto-created on first run вҖ” you don't need to manually create
  the schema. The server runs init_schema at startup to ensure tables exist.

  The database stores:
  - Channels вҖ” tracked/untracked status, Telegram channel metadata
  - Posts вҖ” cached messages (id, channel, UTC timestamp, text)

  Posts older than TGMCPSPY_POST_TTL_DAYS are automatically purged when
  update_channel runs.

  ---
  Quick workflow after launch

  1. sync_dialogs вҖ” pulls your Telegram channel subscriptions into the local
  cache and marks them as tracked
  2. update_all_channels вҖ” fetches recent posts from all tracked channels (first
  time: last 7 days; after: only new posts)
  3. list_channel_posts(channel, start_date, end_date) вҖ” query cached posts by
  date range
  4. add_channel(channel) / remove_channel(channel) вҖ” manage which channels are
  tracked (local only, doesn't change your Telegram subscriptions)

в—Ҹ Running python -m package_tgmcpspy.server starts the MCP server itself вҖ” it
  just listens for MCP protocol connections. It does not launch the MCP
  Inspector.

  To open the Inspector, use the mcp CLI (installed as a dependency via
  mcp[cli]):

  mcp dev src/package_tgmcpspy/server.py

  This will:
  1. Start your MCP server
  2. Automatically open the MCP Inspector in your browser (typically at
  http://localhost:6274)

  From there you can test all the MCP tools interactively.

  If you want to just open the Inspector against an already-running server on
  port 8000, you can also visit https://inspector.modelcontextprotocol.io and
  configure the transport settings manually (SSE URL:
  http://127.0.0.1:8000/sse).

в—Ҹ 1. Add your credentials to .env (in the project root):

  TELEGRAM_API_ID=123456
  TELEGRAM_API_HASH=your_api_hash_here
  TELEGRAM_SESSION_STRING=your_session_string_here

  2. Launch with --env-file flag:

  mcp dev src/package_tgmcpspy/server.py --env-file .env

  The --env-file flag tells mcp dev to pass those variables to the server
  subprocess. Without it, the subprocess starts with a bare environment (as you
  saw in the logs вҖ” no Telegram vars).

  Alternative if --env-file isn't supported by your mcp CLI version вҖ” source it
  into your shell first:

  set -a && source .env && set +a && mcp dev src/package_tgmcpspy/server.py

  set -a auto-exports every variable from .env so the mcp dev subprocess
  inherits them.
